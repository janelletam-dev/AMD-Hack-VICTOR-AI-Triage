## 1. Product Overview

### 1.1 What is V.I.C.T.O.R.?
A voice-first AI triage agent for Emergency Departments that catches cardiovascular disease (CVD) presentations that standard triage misses вҖ” especially in women and minority patients.

### 1.2 The Problem
Atypical cardiovascular presentations kill people because triage systems rely on text-based chief complaints. A woman saying "my stomach hurts and I feel nauseous" gets triaged as ESI 3 (not urgent). A man saying "chest pain" gets ESI 2 (emergent). Both may be having a heart attack вҖ” but only one gets seen quickly.

MIMIC-IV data confirms: abdominal pain in female CVD patients averages acuity 2.8 vs chest pain at 2.15. Same disease, different words, different outcome.

### 1.3 The Solution
Three parallel signals from one voice input:
1. **What the patient says** вҶ’ Deepgram Flux Multilingual (transcript)
2. **How they say it** вҶ’ Thymia Helios + Apollo (voice biomarkers)
3. **What doesn't add up** вҶ’ Concordance Engine + 5-Agent Swarm (bias-aware triage)

### 1.4 Target Users
- **Primary:** ER triage nurses вҖ” see the clinician dashboard
- **Secondary:** Patients вҖ” interact with the voice kiosk
- **Tertiary:** Hackathon judges вҖ” evaluate the demo

## 2. User Stories

### Patient
- As a patient, I want to describe my symptoms by speaking naturally so that I don't have to fill out forms while feeling unwell
- As a non-English-speaking patient, I want the system to understand me in my language so that nothing is lost in translation
- As a patient, I want to choose who I speak with (Victor or Jackie) so that I feel comfortable
- As a patient, I want to confirm the system heard my name and date of birth correctly so that the chart is associated with me

### Clinician
- As a triage nurse, I want to see a real-time transcript of the patient's words so that I have an accurate record
- As a triage nurse, I want to see voice biomarker scores so that I have objective physiological data beyond what the patient tells me
- As a triage nurse, I want to be alerted when the patient's words don't match their voice signals so that I can reassess potential cardiac cases
- As a triage nurse, I want an auto-generated SOAP note so that I can focus on the patient instead of documentation
- As a triage nurse, I want a standard ESI score alongside V.I.C.T.O.R.'s adjusted score so that I can make informed decisions
- As an ER doctor, I want a full evidence report I can push to the chart so that the clinical reasoning is documented

V.I.C.T.O.R.'s clinician view is designed as an EMR module вҖ” not a standalone "AI dashboard." The chrome, tab structure, toolbar icons, and patient banner are intentionally modelled on Epic Hyperspace to reduce adoption friction. Clinicians should feel like they're looking at a new module inside their existing system, not a foreign product.

**Important:** This is mock chrome for demo purposes. V.I.C.T.O.R. is NOT embedded in actual Epic Hyperspace. This should be called out clearly to reviewers so they don't mistake visual fidelity for integration.

### Fine-Tuning Script (Day 3)
```python

from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import LoraConfig, get_peft_model
from trl import SFTTrainer
import torch

# Load base model (ROCm will use AMD GPU automatically)
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Meta-Llama-3.1-8B-Instruct",
    torch_dtype=torch.float16,
    device_map="auto"
)
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Meta-Llama-3.1-8B-Instruct")

# LoRA config
lora_config = LoraConfig(
    r=16, lora_alpha=32,
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    lora_dropout=0.05,
    task_type="CAUSAL_LM"
)

model = get_peft_model(model, lora_config)

# Train
trainer = SFTTrainer(
    model=model,
    train_dataset=dataset,  # loaded from victor_train.jsonl
    max_seq_length=2048,
    args=training_args
)
trainer.train()

# Save and push to Hugging Face

model.push_to_hub("janelletamayo/victor-triage-lora-llama3.1-8b")
```

## 5. 5-Agent Swarm Architecture

All agents run on the same LoRA fine-tuned Llama 3.1 8B model via vLLM on MI300X вҖ” domain-specialised for healthcare triage, different system prompts per agent, same inference endpoint. Falls back to base Llama 3.1 8B Instruct if fine-tuning is incomplete.

### V.I.C.T.O.R. вҖ” Triage Leader
- Orchestrates all agents, routes audio, manages session state
- Makes escalation decisions based on M.E.R.C.E.D. flags
- Adjusts ESI scores based on concordance evidence
- Latency: <500ms decision routing

### J.A.C.K.I.E. вҖ” Patient Voice
- Patient-facing conversational agent (also the female TTS voice)
- Asks clinically relevant follow-up questions in the patient's language
- Shifts to targeted cardiac symptom elicitation when V.I.C.T.O.R. escalates
- Identity-verification flow (name + DOB) is handled deterministically in the frontend before J.A.C.K.I.E. takes over the conversational interview
- Latency: <1s per response

### M.E.R.C.E.D. вҖ” Concordance Analyst
- Silent bias detection (never speaks to patient)
- Runs tiered minimisation dictionary against Thymia biomarker thresholds
- Fires concordance flags with one-sentence clinical gloss
- Latency: <1s per flag

### S.C.R.I.B.E. вҖ” Clinical Note Writer
- Real-time SOAP note generation (updates continuously)

## 7. Concordance Engine Specification

### 7.1 Purpose
Detects when a patient's spoken words diverge from their voice biomarker signals вҖ” the core innovation of V.I.C.T.O.R.

### 7.2 Trigger Logic
```
IF (transcript contains Tier 1 or Tier 2 minimisation phrase)
AND (Thymia Helios stress/distress > threshold OR
     Thymia Apollo anxiety > threshold OR
     Thymia CVD markers elevated)
THEN fire concordance flag вҶ’ M.E.R.C.E.D. generates clinical gloss
     вҶ’ V.I.C.T.O.R. evaluates escalation
```

### 7.3 Minimisation Tiers (from MIMIC-IV analysis)

**Tier 1 вҖ” Highest CVD miss risk (acuity > 2.75):**
- Abdominal pain / stomach pain (acuity 2.80 in female CVD)
- Nausea/vomiting/diarrhea (acuity 2.85-2.91)
- Abd pain with diarrhea (acuity 2.78)

**Tier 2 вҖ” Moderate risk (acuity 2.5вҖ“2.75):**
- Hypertension (2.65)
- Nausea/Vomiting alone (2.63)
- Falls (2.57)

**Tier 3 вҖ” Atypical presentations needing biomarker cross-check:**
- Weakness/fatigue (2.46 in female CVD)
- Epigastric pain / heartburn (2.44)
- Headache (2.69)
- Leg pain (2.83-2.95)

**Tier 4 вҖ” Verbal minimisation patterns:**
- "it's nothing really"
- "I'm fine, just..."
- "probably just stress"
- "I think it's anxiety"
- "probably pulled a muscle"
- "it'll pass"
- "I didn't want to bother anyone"

### 7.4 Clinical Abbreviation Map
The engine translates between patient language and clinical terminology:
- CP вҶ’ Chest pain
- SOB вҶ’ Shortness of breath
- n/v/d вҶ’ Nausea, vomiting, diarrhea
- BRBPR вҶ’ Bright red blood per rectum
- s/p вҶ’ Status post
- ILI вҶ’ Influenza-like illness
- Abd вҶ’ Abdominal

## 15. Out of Scope (Explicitly)

The following are NOT implemented in the hackathon prototype. This section exists so reviewers don't mistake visual fidelity for integration:

- **Real Epic Hyperspace embedding** вҖ” The EMR-style chrome (patient banner, tab structure, toolbar icons) is intentional mock UI to demonstrate how V.I.C.T.O.R. would appear as an Epic module. It is not an actual Hyperspace embedded application.
- **Real EHR push** вҖ” The "Approve & Push to Chart" button demonstrates the workflow but does not connect to any EHR system.
- **Real patient data at runtime** вҖ” All patient demographics in the demo (Maria Hernandez, etc.) are fictional. No MIMIC-IV patient data is displayed.
- **HIPAA/regulatory compliance certification** вҖ” The architecture is designed to be HIPAA-aware (no audio persistence, ephemeral rooms) but has not been formally audited or certified.
- **Clinical validation trial** вҖ” V.I.C.T.O.R. is a proof of concept. The bias detection logic is evidence-informed (MIMIC-IV) but has not been validated in a clinical setting.
- **Production-grade error handling** вҖ” Edge cases (network drops, API failures, concurrent sessions) are handled at demo level, not production level.

## 16. Risks & Mitigations
